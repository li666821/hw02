# hw02
利用大模型生成学术论文导读，以及通过API调用大模型完成对话式应用。编写一个可运行的Chat bot实例代码，体验从读论文到用模型的完整链路。
（1）
一、论文来源
该论文为宋志强、周鹏、宋利鹏所著的《基于多模态融合的行人轨迹预测方法》，发表于《计算机应用与软件》期刊。.
二、导读生成方式
以论文原文核心内容为唯一依据，按照研究背景与动机→核心方法→主要结果→个人小结的逻辑结构逐层拆解创作：
1. 研究背景与动机：提炼传统行人轨迹预测方法的三大核心痛点（长时预测误差累积、无全局目标约束、CVAE潜变量建模能力不足），明确研究的问题导向；
2. 核心方法：聚焦论文提出的目标条件多模态双向轨迹预测模型，按“潜变量增强→全局目标约束→双向轨迹生成→损失函数优化”的模型设计逻辑，拆解CVAE+扩散模型、目标点生成器、双向解码器、多维度损失函数四大核心设计；
3. 主要结果：严格依据论文JAAD、PIE数据集的实验数据，从整体性能、消融实验、可视化结果三方面总结，如实呈现定量指标（如PIE数据集NLL降至15.6、JAAD长期ADE降至199）与定性结论；
4. 个人小结：结合模型创新点（CVAE与扩散模型融合、目标引导双向解码）总结研究价值，同时呼应论文提及的局限性，梳理后续优化方向，形成“创新-价值-局限-展望”的总结逻辑。
三、配图方式
围绕导读四大板块及个人小结，引用的专题配图，配图均贴合论文核心内容，采用“模块架构/数据可视化/场景对比/框架总结”的形式，与导读内容一一对应。
所有配图均遵循“贴合论文原文、匹配导读逻辑、可视化核心信息”的原则，无额外拓展，仅对导读中的关键内容进行视觉化呈现。
（2）
### 采用的API/平台
- 使用DeepSeek的OpenAI兼容接口
- API基础URL: https://api.deepseek.com/v1
- 模型: deepseek-chat
### 运行环境
- Python 3.7+
- Windows 10/11 或其他支持Python的操作系统
### 依赖安装
```bash
# 安装openai库
pip install openai

# 或使用requirements.txt
pip install -r requirements.txt
```
### 如何配置API Key
为了提高安全性，本示例采用在运行时交互式输入API密钥的方式，避免在代码中硬编码：

1. 运行脚本后，会提示你输入API密钥
2. 输入时，密钥会被隐藏（不会显示在屏幕上）
3. 输入完成后按回车键确认
这种方式可以有效避免API密钥泄露的风险，适合在本地开发和测试环境中使用。
### 运行命令
```bash
# 默认发送"你好"消息
python chatbot_deepseek.py
# 发送自定义消息
python chatbot_deepseek.py 你的问题
# 示例
python chatbot_deepseek.py 你是谁
python chatbot_deepseek.py 什么是机器学习
```

### 注意事项
1. **API密钥获取**：请确保你已经从DeepSeek官网获取了有效的API密钥
2. **API费用**：API调用可能会产生费用，请根据DeepSeek的 pricing 计划了解详情
3. **网络连接**：确保你的网络连接正常，能够访问DeepSeek API服务
4. **环境变量问题**：如果之前设置过 `DEEPSEEK_API_KEY` 环境变量，可能会影响脚本运行，请确保该环境变量未被设置为错误值
5. **脚本修改**：如需使用其他模型或API，请修改脚本中的 `_BASE_URL` 和 `model` 参数
6. **响应时间**：API调用可能需要几秒钟时间，请耐心等待
7. **错误处理**：如果遇到认证失败等错误，请检查API密钥是否正确，以及是否有足够的余额
8. **结果输出**：脚本会将完整的对话结果写入 `result.txt` 文件，以便查看完整输出
### 故障排除
- **认证失败错误**：请检查API密钥是否正确，是否有足够的余额
- **网络错误**：请检查网络连接，确保能够访问DeepSeek API服务
- **环境变量冲突**：请确保没有设置错误的 `DEEPSEEK_API_KEY` 环境变量
